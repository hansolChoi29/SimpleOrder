### 주문 목록 조회 시 N+1 문제 해결

Order와 Product가 @ManyToOne(fetch = FetchType.LAZY) 관계라서, 기존 findAll()로 Order 목록을 가져온 뒤 order.getProduct()를 호출하면 Order 개수만큼 Product SELECT 쿼리가 추가로 발생했습니다.


이를 해결하기 위해 OrderRepository에 @Query로 JOIN FETCH 쿼리를 직접 작성했습니다. JOIN FETCH는 Order 조회 시 Product를 처음부터 함께 JOIN해서 한 번의 쿼리로 가져오기 때문에 추가 쿼리가 발생하지 않습니다. 페이지네이션은 Pageable을 파라미터로 받아 동일한 쿼리에서 함께 처리했습니다.


```
select
    o1_0.id, o1_0.created_at, o1_0.product_id,
    p1_0.id, p1_0.name, p1_0.price, p1_0.status, p1_0.stock_quantity,
    o1_0.quantity, o1_0.status, o1_0.total_price
from orders o1_0
join products p1_0 on p1_0.id = o1_0.product_id
limit ?
```
