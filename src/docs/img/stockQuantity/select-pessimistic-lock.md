### 상품 재고 원자적 차감 — 비관적 락(Pessimistic Lock) 선택 이유

주문 생성 시 재고 조회부터 차감까지의 흐름은 원자적으로 처리되어야 합니다. 

낙관적 락(Optimistic Lock)은 충돌이 발생한 후 예외를 던지고 재시도를 애플리케이션에서 직접 처리해야 하는 반면, 비관적 락은 SELECT ... FOR UPDATE로 조회 시점에 DB 레벨에서 행(row)을 잠가 다른 트랜잭션이 접근 자체를 못 하게 막습니다.


이번 프로젝트처럼 재고가 소진되면 주문이 실패해야 하는 구조에서는, 충돌 후 복구보다 충돌 자체를 차단하는 비관적 락이 더 적합하다고 판단했습니다. 

ProductRepository에 @Lock(LockModeType.PESSIMISTIC_WRITE)를 적용해 주문 생성 트랜잭션 내에서 해당 상품 row를 잠근 뒤 재고를 차감하고 커밋하는 방식으로 구현했습니다.
